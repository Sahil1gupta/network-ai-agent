# 🔴 STC AutoOSS — AI NOC Copilot

> Automated fault detection, Root Cause Analysis, and incident ticket management powered by AI agents and RAG.

Built for **TCS AI Friday Hackathon** | Client: **STC Saudi Arabia** | Team: **STC AutoOSS**

---

## 📌 Problem Statement

Traditional NOC (Network Operations Center) operations at STC require engineers to manually analyze network alarms, identify root causes, and create incident tickets — a process that takes **45+ minutes** per incident.

**STC AutoOSS** automates this end-to-end using a multi-agent AI pipeline, reducing MTTR to under **8 minutes** and auto-resolving **80% of L1 tickets** without human intervention.

---

## ✨ Features

- ⚡ **Multi-Agent AI Pipeline** — 4 specialized agents process each alarm sequentially
- 🧠 **RAG (Retrieval Augmented Generation)** — AI answers using STC-specific runbooks, not generic knowledge
- 📋 **Auto Ticket Generation** — structured incident tickets with RCA, priority, and resolution steps
- 📧 **Email Notifications** — L1 team notified instantly via Gmail SMTP
- ⏱️ **Auto Escalation** — background timer escalates unresolved tickets to L3 team automatically
- 📊 **Live Dashboard** — Streamlit UI with real-time metrics and ticket management
- 📥 **Export** — download all tickets as JSON

## 🏗️ Architecture

```
User selects alarm → clicks Analyze
        ↓
    app.py (Streamlit)
        ↓
    pipeline.py — orchestrates 4 agents
        ↓
┌──────────────────────────────────────────┐
│  Agent 1: Fault Classifier               │
│  → OpenAI GPT-4o classifies fault type  │
│  → Returns: priority P1/P2/P3, urgency  │
├──────────────────────────────────────────┤
│  Agent 2: Context Fetcher (RAG)          │
│  → ChromaDB similarity search            │
│  → Returns: relevant runbook text        │
├──────────────────────────────────────────┤
│  Agent 3: RCA Generator                  │
│  → OpenAI GPT-4o + runbook context      │
│  → Returns: root cause, fix steps        │
├──────────────────────────────────────────┤
│  Agent 4: Ticket Creator                 │
│  → Assembles final incident ticket       │
│  → Returns: complete ticket dict         │
└──────────────────────────────────────────┘
        ↓
┌─────────────────────────────────┐
│  3 things happen simultaneously  │
│  1. Dashboard UI updated         │
│  2. L1 team email sent           │
│  3. Escalation timer started     │
└─────────────────────────────────┘
        ↓
If not resolved in X minutes → L3 escalation email auto-sent
```

## 🚀 Quick Start

### 1. Clone and Setup

```bash
git clone <repo-url>
cd stc-autooss

python3 -m venv venv
source venv/bin/activate        # Mac/Linux
# venv\Scripts\activate         # Windows
```

### 2. Install Dependencies

```bash
pip install openai langchain langchain-openai langchain-community \
            chromadb streamlit python-dotenv faker
```

### 3. Configure Environment

Create `.env` file in project root:

```env
OPENAI_API_KEY=sk-xxxxxxxxxxxxxxxxxxxxxxxx

GMAIL_USER=your_email@gmail.com
GMAIL_APP_PASS=xxxx xxxx xxxx xxxx

L1_EMAIL=l1team@gmail.com
L3_EMAIL=l3team@gmail.com
```

> **Gmail App Password:** myaccount.google.com → Security → 2-Step Verification → App Passwords → Generate

### 4. Generate Synthetic Data

```bash

# Output: data/alarms.json (60 alarms)

```

### 5. Test Connection

```bash

# Output: STC AutoOSS is ready!
```

### 6. Launch Dashboard

```bash
streamlit run src/app.py
# Opens: http://localhost:8501
```

## 🔧 Component Details

### `src/llm_client.py`
Central OpenAI connection file. All other modules import from here.

```python
get_llm(model="gpt-4o")          # chat model
get_embeddings()                   # text-embedding-3-small
```

### `src/rag_pipeline.py`
Builds and queries ChromaDB vector store from runbook `.txt` files.

- `build_vector_db()` — run once at startup
- `retrieve_context(query, k=2)` — returns top-k similar runbook chunks

### `src/agents.py`
Four specialized agents:

| Agent | Function | LLM Used |
|-------|----------|----------|
| Agent 1 | `classify_fault(alarm)` | GPT-4o |
| Agent 2 | `fetch_context(alarm, classification)` | ChromaDB |
| Agent 3 | `generate_rca(alarm, classification, context)` | GPT-4o |
| Agent 4 | `create_ticket(alarm, classification, rca)` | Pure Python |

### `src/email_service.py`
Gmail SMTP email sender.

- `send_new_ticket_email(ticket)` — L1 team notification
- `send_escalation_email(ticket)` — L3 escalation alert

### `src/escalation_service.py`
Background thread-based escalation timer.

- `start_timer(ticket, minutes=2)` — starts countdown
- `mark_resolved(ticket_id)` — cancels escalation
- `get_status(ticket_id)` — returns OPEN / RESOLVED / ESCALATED
